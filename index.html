Clan Sync

Set Up Shared Sync

Clan Sync is optional. It creates one shared online space for dragons and map pins that members deliberately share. Your regular tracker data, backups, account passwords, login codes, and Steam Guard information always stay out of it.

Starting a Clan

I am the organizer

You need one trusted person to maintain the shared sync space. This can be you or another clan organizer. Everyone else can join without doing this setup.

  1. Create the clan's free Supabase project. Supabase is the service that provides the private online shelf holding only the dragons and pins members choose to share.
  2. Use the organizer checklist below to prepare that shelf and connect Discord sign-in. You will need a free Discord developer application for that one-time organizer task.
  3. Collect the project's Sync address and public Connection key. Give those two items to people who should join this shared space.
  4. Choose Connect Sync below, paste those two items, connect Discord, create the clan, and make one-use invite codes for members.

The project owner should keep its administrator password, secret keys, and Discord app secret private. Clan members never need any of them.

Organizer checklist before inviting members
  1. Create a free Supabase project. In its Settings > API area, copy the Project URL and its public anon or publishable key. Those become the clan's Sync address and Connection key.
  2. Open Supabase's SQL Editor. Run Dragon Tracker's supplied clan-sync setup file, supabase/migrations/0001_clan_sync.sql. It creates the private clan tables and access rules.
  3. Create a Discord application in the Discord Developer Portal. In Supabase Authentication, enable Discord, add the Discord Client ID and Client Secret, and request only the identify permission.
  4. In the Discord application, add https://YOUR_PROJECT_REF.supabase.co/auth/v1/callback as the callback address. Replace YOUR_PROJECT_REF with the project reference from your Supabase URL.
  5. In Supabase Authentication's allowed redirect addresses, add dragontracker://auth/callback for the installed app. Add the exact local-browser address too, such as http://127.0.0.1:8765/index.html, if members will use it.
  6. Connect sync and Discord in Dragon Tracker using a test account, create a test clan, then verify that a second Discord account can join with a one-use invite and view one deliberately shared test dragon.

Joining a Clan

My organizer set it up

Ask an organizer for these three things:

  • Sync address (also called the Project URL)
  • Connection key (also called the public anon key)
  • One-use clan invite code
  1. Choose Connect Sync. Paste the organizer's sync address into Sync address. It starts with https:// and belongs to the clan's shared space, not your own computer. Paste the second item into Connection key, then save.
  2. Open Clans and choose Connect Discord. Dragon Tracker opens Discord in your normal browser. Approve the identity screen, then return to the tracker when Discord finishes. Discord confirms your tracker identity; it does not give Dragon Tracker access to messages, servers, email, or your password.
  3. Paste the one-use invite code to join. Only then can you see the clan's shared dragons and pins.
  4. Use Share to Clan on a dragon or map pin when you want other clan members to see it. Nothing is shared automatically.

Desktop app and browser-local tracker: both use the same sync address, connection key, Discord identity, and invite code. For the browser-local tracker, open it through http://localhost or http://127.0.0.1, not by double-clicking an index.html file. The organizer must add that exact local address to the shared space's allowed return addresses before you connect Discord.

Optional: add dragons from a Discord server

A Discord bot should submit proposed shared dragons to the clan's secure backend. It must never edit a member's local tracker automatically, read DMs, or store Discord passwords, Steam credentials, backup files, or login codes.

  1. The clan organizer creates a separate Discord application and bot. Install it only in the clan's server, with only the command permissions it needs. Do not give it Administrator.
  2. Create a slash command such as /dragon submit. Have it ask for the dragon's display name, species, skin, recessive skin, stage, and optional public notes.
  3. Point Discord's Interactions Endpoint at a server-side function, such as a Supabase Edge Function. That function must verify every Discord request signature before accepting data.
  4. Keep the bot token, Discord application secret, Discord public-key verification settings, and Supabase service-role key in the host's server secrets. They never belong in Dragon Tracker, a backup, GitHub source, or a clan chat.
  5. Make every submission pending. A clan owner or admin reviews it before it becomes a shared dragon. The bot should save only the approved shared summary, never accounts or private notes.

This is an optional hosted integration and is not connected automatically in this build. It needs a dedicated server-side function and Discord bot setup per clan. The normal in-app Share to Clan button already works without a bot.
